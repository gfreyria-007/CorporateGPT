## Architecture Decisions

### AI Engine Routing
- All AI requests route through OpenRouter via backend proxy (`/api/chat`, `/api/techie`, `/api/fallback`)
- "Immortal Architecture": multi-model failover with prioritized model chains
- Default model: `openrouter/auto` (auto-routes to best available)
- Both Corporate GPT and Techie share the same model list from `/api/models`
- User can select specific models via `ModelSelector` component
- **Techie standardized**: All client-side SDKs (`GoogleGenAI`) and hardcoded keys removed. Techie routing is strictly proxy-only via `/api/techie`.

### Techie Workspace
- Techie is a child workspace inside Corporate GPT, rendered via `TechieWorkspace.tsx` bridge
- Entry point: `src/Techie/src/TechieMain.tsx`
- Service layer: `src/Techie/src/services/geminiService.ts` (now 100% proxy-based)
- Techie uses the same backend proxy `/api/techie` which inherits the Immortal Architecture failover
- `getChatResponse` accepts optional `selectedModel` parameter (defaults to `openrouter/auto`)
- **UI Standardized**: `ModelSelector` integrated into `ChatInput` area, replacing the "Cosas Pro" legacy label.
- **Mascot**: Local asset `/techie-mascot.png` replaces broken hardcoded external link.
- MonthlyQuotaBanner removed from Techie (not kid-appropriate)
- Responsive Layout optimized for `h-screen overflow-hidden`.


### Responsive Layout (Techie)
- Root container: `h-screen overflow-hidden` (not min-h-screen)
- Chat area: `flex-1 flex flex-col min-h-0` with `overflow-y-auto` inner div
- Header/footer: wrapped in `shrink-0`
- Top controls (grade + model selector): single compact row with `flex-wrap`
